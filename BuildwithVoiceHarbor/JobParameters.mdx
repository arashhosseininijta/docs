## Parameters

### Audio Upload and Token Authentication

When submitting your job via API, ensure the following:
- The `source_audio` is provided via `multipart/form-data`
- The `developer_token` must be included in the `Authorization` header as a Bearer token

```bash
-H "Authorization: Bearer YOUR_USAGE_TOKEN"
```

<ResponseField name="biometric-age" type="string" default="young-adult">

Allowed values:
- same

<ResponseField name="language" type="string" default="en">
Language for speaker pool matching or avatar voice.
Allowed values:
- en
- fr

- es
- de
</ResponseField>

<ResponseField name="model_id" type="string" default="mini">
Model variant for biometric voice print redaction.
Allowed values:
- mini
- advanced
</ResponseField>

<ResponseField name="avatar" type="string" optional>
Optional avatar identity to apply instead of biometric-age/gender.
‚ö†Ô∏è Do not supply `biometric-age` or `gender` when using `avatar`. Only works with language: en, fr, it.

<ResponseField name="emotion" type="string" optional>
Optional emotion to apply to avatar identity.
‚ö†Ô∏è Only used with `avatar`. Must match one of the supported emotions for that avatar.
</ResponseField>

### Example Curl Usage

#### ‚úÖ Minimal biometric redaction request (uses defaults)

```bash
curl -X POST https://voiceharbor.ai/api/speech-to-speech-biometric-redaction \
  -H "Authorization: Bearer YOUR_USAGE_TOKEN" \
  -H "Content-Type: multipart/form-data" \
  -F "source_audio=@./martin.wav" \
  --output result.wav
```

#### üéØ Custom biometric request

```bash
curl -X POST https://voiceharbor.ai/api/speech-to-speech-biometric-redaction \
  -H "Authorization: Bearer YOUR_USAGE_TOKEN" \
  -H "Content-Type: multipart/form-data" \
  -F "source_audio=@./martin.wav" \
  -F "model_id=advanced" \
  -F "biometric-age=same" \
  -F "gender=opposite" \
  -F "language=fr" \
  --output redacted_fr.wav
```

#### üë§ Avatar redaction with emotion

```bash
curl -X POST https://voiceharbor.ai/api/speech-to-speech-biometric-redaction \
  -H "Authorization: Bearer YOUR_USAGE_TOKEN" \
  -H "Content-Type: multipart/form-data" \
  -F "source_audio=@./martin.wav" \
  -F "avatar=emma" \
  -F "emotion=happy" \
  -F "language=en" \
  --output avatar_emma_happy.wav
```

## ‚ö†Ô∏è Possible API Errors

| HTTP Code | Error Condition | Message |
| --------- | --------------- | ------- |
| 400       | Invalid field value or combination | Descriptive error string |
| 401       | Missing or malformed Authorization header | `"Missing Authorization header"` |
| 403       | Invalid developer token | `"Invalid token"` |
| 400       | `avatar/emotion` with unsupported language | `"avatar/emotion only supported when language=en, fr, it"` |
| 400       | Mixed biometric and avatar params | `"avatar and emotion cannot be used together with biometric-age or gender"` |
| 400       | Missing audio | `"Missing audio file"` |
| 400       | Invalid model_id | `"Invalid model_id. Allowed: mini, advanced"` |
| 400       | Invalid biometric-age | `"Invalid biometric-age. Allowed: young-adult, middle-aged-adult, senior-adult, same, random"` |
| 400       | Invalid gender | `"Invalid gender. Allowed: same, random, opposite"` |
| 400       | Invalid avatar or emotion | `"Invalid avatar/emotion. Allowed: ..."` |
| 500       | Backend failure | `"Proxying failed"` |
| 503       | Model not loaded | `"Model not loaded"` |
